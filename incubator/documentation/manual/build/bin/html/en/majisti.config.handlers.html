1.2. Handlers

Since Majisti Application is built solely on configuration, some more config handling was needed. Reading configuration is the responsability of Zend_Config and therefore no other tasks should be defined on how that read config is interpreted. Therefore, with config handlers, it is possible to parse specific configuration syntax. The following are the default configuration handlers shipped with Majisti.

1.2.1. Property

The property handler listens on the properties attribute just like apache ant, consider the following application.ini example: 

[production]

properties.applicationUrl = APPLICATION_URL
properties.baseUrl = "#{applicationUrl}MyProject/public"
properties.imagesUrl = #{baseUrl}/images
properties.scriptsUrl = #{baseUrl}/scripts
properties.locale = en

images.thumbsUp = #{imagesUrl}/thumbsUp.png

Every occurences will get replaced with their proper properties.
